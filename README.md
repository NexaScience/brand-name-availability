# brand-name-availability

AIサービス等の **サービス名/ブランド名候補** を、サブエージェントで並列調査して
候補ごとに ○△× のリスク表にまとめる Claude Code スキル。

## 調べる6軸

| # | 軸 | 内容 |
|---|---|---|
| 1 | 日本商標 | J-PlatPat で同一・類似の登録/出願（DOMテキスト読み） |
| 2 | 米国商標 | USPTO の Live/Dead と区分（API or DOMテキスト） |
| 3 | ドメイン | .ai/.io/.com の空き＋価格（RDAP/WHOIS） |
| 4 | SNS・実在 | ハンドル空き＋同名サービスの実在 |
| 5 | 語感（ネガティブ） | 他言語/他地域で悪い意味に一致・酷似しないか |
| 6 | 発音品質 | 読みやすさ・初見での発音認識・音の美しさ・人工感 |

## 方針

- 構造化データ（API）優先。ブラウザは画像でなく `get_page_text` で DOM テキストを読む。
- 米国商標は第三者API→公式TSDRの2段で自動化可（無料枠あり）。
- 日本商標は規約上、対話的なブラウザDOM読みが現実解。
- **本スキルは事前スクリーニング。最終判断は弁理士に確認すること。**

## 使い方

`SKILL.md` を Claude Code のスキルとして配置（例: `~/.claude/skills/brand-name-availability/SKILL.md`）。
「この名前使える？」「商標とドメイン取りたい」等で起動する。
